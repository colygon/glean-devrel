# Glean DevRel KPIs & Success Metrics

## Executive Summary

This document defines key performance indicators (KPIs) for Glean's Developer Relations program. Metrics are organized into three tiers: **North Star Metrics** (business impact), **Program Metrics** (initiative health), and **Activity Metrics** (operational execution).

**Philosophy:** DevRel success is measured by community health, developer adoption, and ecosystem growthâ€”ultimately driving product improvement and business outcomes.

### Real-World Benchmarks

Based on proven DevRel results from similar programs:
- **70 projects delivered** (41 bounties paid)
- **10,000+ views** on educational content
- **3 successful hacker houses** (32 guests, 12 days)
- **80% completion rate** (last cohort)

These benchmarks inform our targets and demonstrate achievability.

## North Star Metrics

### 1. Active Community Members
**Definition:** Developers who engage with Glean DevRel programs monthly

**Measurement:**
- Discord active users (posted/reacted in last 30 days)
- Workshop/meetup attendees
- Bounty/grant participants
- MCP server publishers
- Hacker house residents

**Targets:**
- Month 3: 500
- Month 6: 2,000
- Month 12: 10,000
- Year 2: 25,000

**Why It Matters:** Community size indicates reach and engagement potential

---

### 2. Developer Adoption Rate
**Definition:** New Glean users attributed to DevRel efforts

**Measurement:**
- Sign-ups with DevRel referral codes
- Workshop â†’ Installation conversion
- Community-driven installations (tracked via UTM)
- MCP server installs â†’ Glean downloads

**Targets:**
- Month 3: 500 new users
- Month 6: 2,000 new users
- Month 12: 10,000 new users
- Year 2: 50,000 new users

**Why It Matters:** DevRel should drive measurable product adoption

---

### 3. Ecosystem Contribution Value
**Definition:** Value created by community (code, content, integrations)

**Measurement:**
- MCP servers published (weighted by downloads)
- GitHub contributions to Glean ecosystem
- Tutorials/content created (weighted by views)
- Bounties completed ($)
- Extensions published

**Targets:**
- Month 3: 50 contributions
- Month 6: 200 contributions
- Month 12: 1,000 contributions
- Year 2: 5,000 contributions

**Why It Matters:** Healthy ecosystem reduces Glean's engineering burden and increases product value

---

### 4. Developer Retention (30/60/90 Day)
**Definition:** % of DevRel-acquired users still active after 30/60/90 days

**Measurement:**
- Track cohorts from DevRel sources
- Monitor Glean usage (logins, commands run)
- Survey at 30/60/90 day marks

**Targets:**
- 30-day retention: 70%
- 60-day retention: 55%
- 90-day retention: 45%

**Why It Matters:** Quality > quantity; retained users become advocates

---

### 5. Net Promoter Score (NPS)
**Definition:** Community satisfaction and likelihood to recommend Glean

**Measurement:**
- Quarterly NPS surveys
- Post-event NPS
- Community sentiment analysis

**Targets:**
- Month 6: NPS 40+
- Month 12: NPS 50+
- Year 2: NPS 60+

**Why It Matters:** Happy developers become evangelists and drive word-of-mouth growth

### Developer Journey Tracking

**Acquisition â†’ Activation â†’ Retention â†’ Referral**

1. **Acquisition**: How did they find Glean?
   - DevRel event/content
   - Community referral
   - Organic search
   - Paid marketing

2. **Activation**: First meaningful action
   - Install Glean
   - Complete first workflow
   - Join Discord
   - Attend workshop

3. **Retention**: Ongoing engagement
   - Weekly active usage
   - Return to events
   - Contribute to ecosystem

4. **Referral**: Advocacy
   - Refer others
   - Create content
   - Speak at events
   - Contribute code

## Optimization & Iteration

### Quarterly Review Process

**Week 1: Data Collection**
- Compile all metrics
- Gather qualitative feedback
- Interview top contributors

**Week 2: Analysis**
- Identify trends (positive/negative)
- Compare against targets
- Calculate ROI per program

**Week 3: Planning**
- Set next quarter goals
- Allocate budget adjustments
- Plan new initiatives

**Week 4: Communication**
- Present QBR to leadership
- Share wins with community
- Update public roadmap

### A/B Testing Approach

Test and optimize:
- Workshop times (weekday vs weekend)
- Content formats (live vs recorded)
- Prize structures (cash vs credits)
- Event lengths (1hr vs 2hr vs 4hr)
- Marketing channels (Twitter vs Reddit vs HN)

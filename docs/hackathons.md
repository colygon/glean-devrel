# Glean Quarterly Hackathons

## Overview

Join our quarterly hackathons to build innovative tools, extensions, and integrations for Glean. Compete for larger prizes, get intensive mentorship, and showcase your projects to the community.

## Hackathon Schedule

### Frequency
- **Quarterly**: Once per quarter (4 per year)
- **Duration**: 48 hours (Friday 6pm PT - Sunday 6pm PT)
- **Format**: Virtual + optional in-person (SF)

### 2026 Schedule

| Quarter | Dates | Theme |
|---------|-------|-------|
| Q1 2026 | Feb 28 - Mar 2 | MCP Ecosystem Building |
| Q2 2026 | May 30 - Jun 1 | AI & Automation |
| Q3 2026 | Aug 29-31 | Developer Productivity |
| Q4 2026 | Nov 21-23 | Year-End Innovation Showcase |

## How It Works

### Before the Hackathon (1-2 weeks prior)

**1. Registration Opens**
- Sign up individually or as a team (max 4 people)
- Choose your category
- Review theme and challenges

**2. Preparation**
- Join Discord #hackathon channel
- Review starter kits and resources
- Form teams (optional team matching)
- Set up development environment

### During the Hackathon

**Friday Evening: Kickoff (6pm - 8pm PT)**
- Opening ceremony livestream
- Theme announcement and challenge details
- Q&A with judges
- Form or finalize teams
- Start building!

**Saturday: Build Day (All day)**
- Build your project
- Mentorship hours (10am - 6pm PT)
- Optional check-ins
- Mini-workshops

**Sunday: Final Push & Demos (Until 6pm PT)**
- Complete your project
- Submit by 6pm PT sharp
- Prepare 3-minute demo video
- Finalize documentation

**Sunday Evening: Judging & Awards (7pm - 9pm PT)**
- Finalist demos (live presentations)
- Judge deliberation
- Winner announcements
- Prizes awarded

## Judging Criteria

Projects evaluated on:

1. **Innovation** (25%)
   - Originality of idea
   - Creative problem-solving
   - Unique approach

2. **Technical Implementation** (25%)
   - Code quality
   - Architecture design
   - Technical difficulty

3. **Usefulness** (20%)
   - Practical value
   - Solves real problems
   - User benefit

4. **Polish** (15%)
   - UI/UX design
   - Documentation
   - Demo quality

5. **Theme Alignment** (15%)
   - Fits monthly theme
   - Uses required technologies

## Submission Requirements

### Required Deliverables

1. **Working Demo**
   - Deployed or demo-able project
   - Public GitHub repository
   - Installation instructions

2. **Demo Video** (3 minutes max)
   - Show the project in action
   - Explain the problem & solution
   - Technical highlights

3. **Documentation**
   - README with overview
   - Setup/installation guide
   - Technical architecture
   - Team information

4. **Presentation Deck** (Optional)
   - For finalists only
   - 5-7 slides

## FAQ

**Q: Can I work on an existing project?**
A: No, projects must start during the hackathon kickoff.

**Q: Can I participate solo?**
A: Absolutely! Solo participants are welcome.

**Q: What if I can't finish in time?**
A: Submit what you have! Judges value progress and effort.

**Q: Can I use third-party libraries?**
A: Yes! Use any open-source libraries you need.

**Q: Do I retain IP rights?**
A: Yes, you own your code. We only require it to be open source.

**Q: Can I continue working after the hackathon?**
A: Yes! Many projects become ongoing open-source projects.

**Q: What if my team member drops out?**
A: You can continue solo or find replacement via Discord.

**Q: Is internet access required?**
A: Yes, the hackathon is online and requires internet access.
